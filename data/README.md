# Data

The main dataset required is `yourground_ml_features.csv`.

## How to obtain it

This file is generated by the data preparation pipeline and is **not included** in the repository due to its size (~4MB) and the API keys required to generate it.

### Option A — Contact the author
Reach out to request the pre-generated CSV directly.

### Option B — Generate it yourself
Follow the full pipeline in order:

1. **Download YourGround reports** from the YourGround platform (Melbourne safety reports dataset)

2. **Run Gemini Vision analysis** — scores 29 visual features from Google Street View images for each report location using the Gemini 2.5 Flash API:
   ```
   python data_preparation/14_full_dataset_analysis.py
   ```
   Requires: `GEMINI_API_KEY` and `GOOGLE_MAPS_API_KEY` environment variables

3. **Compute space syntax measures** — downloads Melbourne OSM street network and computes integration, mean depth, edge length, choice for each location:
   ```
   python data_preparation/15c_space_syntax_fast.py
   ```

4. **Add OSM land use zones** — fetches land use polygons (residential, commercial, parkland, etc.) and spatially joins them to each report:
   ```
   python data_preparation/17_add_land_use.py
   ```

## Dataset columns

Once generated, `yourground_ml_features.csv` contains:

| Column group | Count | Description |
|---|---|---|
| Original YourGround columns | ~80 | Survey responses, location, time, stress rating |
| Gemini visual features | 29 | Scored 0.0–1.0 from Street View images |
| Space syntax (primal graph) | 7 | ss_connectivity, ss_integration, ss_choice, ss2_* |
| Zone type (OSM land use) | 7 | zone_residential, zone_commercial, zone_parkland, etc. |
| Engineered targets | 5 | stress_num, stress_binary, is_after_dark, has_good_light, has_poor_light |

**Key target columns:**
- `stress_num` — original stress rating 0–5
- `stress_binary` — 0 (stress < 3) or 1 (stress >= 3)
